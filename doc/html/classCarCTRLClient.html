Detailed Description

This class is the client that controls the car.

This class handles the VSOMEIP communication between nodes. It subscribes to a number of event publishers and requests association with the steering service and the motor service. For communicating with the higher-level library ESSPrototype it uses a number of shared memory locations to write and read sensor values and requests, respectively. It is worth noting that ESSPrototype can be replaced with some other software as long as that software still writes to the same shared memories.

◆ update_go_status()

void CarCTRLClient::update_go_status ( )
private

Handler for bringing up/down the go-server.

This member is called by on_availability everytime the availability of services on the network changes. Depending on what is available it either brings up or down the go-server and sets go_ to the corresponding value. It also communicates with the ESSPrototype class over a shared memory so ESSPrototype can see the availability of services as well.

◆ go_

bool CarCTRLClient::go_
private

Pause switch for all threads.

All threads will just loop in one place without doing anything if set to false. Set to true when a remote server (the go-server) comes online. Set to false if the go-server goes offline or if stop is called. Initial value is set to false. This class is in charge if bringing the go-server up/down.
