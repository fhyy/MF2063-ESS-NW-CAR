Detailed Description

High-level library for writing car programs.

This library aims to provide the developer with a set of high-level functions which the developer can use to build programs that control the car, either autonomously or manually. A very simple example of such a program is the ESS Shell (bbb-car-controller/src/ess_shell.cpp).

Author
Leon Fernandez

Constructor & Destructor Documentation

◆ ESSPrototype()

ESSPrototype::ESSPrototype ( )

Constructor for the ESSPrototype class.

An object of the ESSPrototype class is meant to represent a self-aware car. It can do basic car things such as: set the speed to a certain level, change the angle of the wheels and check how fast it is going. But it is also aware of its internal conditions in the sense that it can detect if any of the basic services goes missing. For instance if the power to the camera node is cut, an object of this class has the ability to detect this.

Returns
The free distance ahead.

◆ getGoStatus()

bool ESSPrototype::getGoStatus ( )

Method for checking if the car is ready to go or not.

Returns
True if all vsomeip services are up, false otherwise

Loads the latest status of all vsomeip services in the form of a one-hot encoded int. It is important to know that checkMotor, checkSteering checkSpeedSensor, checkCameraSensor and checkDistanceSensor WILL NEVER UPDATE THEIR RETURN VALUE UNLESS THIS FUNCTION IS CALLED BETWEEN INVOCATIONS.

◆ service_status_

int ESSPrototype::service_status_
private

A one-hot encoded int where every bit represents the availability of a service.

This member is updated every time

See also
CarCTRLClient::update_go_status is called and it holds the information of what services where available at the time. checkMotor, checkSteering checkSpeedSensor, checkCameraSensor and checkDistanceSensor only read from this int when they are called, which means that
CarCTRLClient::update_go_status has to be called in order to update the output of the check-methods.
